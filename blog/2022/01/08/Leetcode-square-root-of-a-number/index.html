[Leetcode][69. Sqrt(x)] 4 Approaches for Finding the Square Root of A Number: Brute Force, Exponent, Binary Search and the Newton Iteration Method

By Long Luo

This article is the solution 4 Approaches: Brute Force, Exponent, Binary Search and the Newton’s Iteration Method of Problem 69. Sqrt(x).

Here shows 4 approaches for finding the square root of a number: Brute Force, Exponent, Binary Search and the Newton’s Iteration Method.

Given an integer NNN and a tolerance level LLL, the task is to find the square root of that number.

Brute Force

The Brute Force way is very easy, just enumerate a value from 000 to xxx, check the product i2i^2i2 and target, return the answer.

Analysis

  • Time Complexity: O(n)O(n)O(n)
  • Space Complexity: O(1)O(1)O(1).

Exponent

Noted that:

x=x1/2=(eln⁡x)1/2=e12ln⁡x\sqrt{x} = x^{1/2} = (e ^ {\ln x})^{1/2} = e^{\frac{1}{2} \ln x} x​=x1/2=(elnx)1/2=e21​lnx

So we can use the exponent exp⁡\expexp and logarithm ln⁡\lnln to calculate the square root of the number x\sqrt{x}x​.

It’s really a fast and simple way!

Note: Since the computer can’t store the exact value of the float number, and the parameters and return values of the exponential function and logarithmic function are float numbers, so the result may be wrong.

For example, when x=2147395600x = 2147395600x=2147395600, the result of e12ln⁡xe^{\frac{1}{2} \ln x}e21​lnx is 10−1110^{-11}10−11 from the correct value of 463404634046340, so when taking the integer part of the result, you will get the wrong result of 463394633946339.

So after getting the integer part ans\textit{ans}ans of the result, we should find out which of ans\textit{ans}ans and ans+1\textit{ans} + 1ans+1 is the real answer.

Analysis

  • Time Complexity: O(1)O(1)O(1)
  • Space Complexity: O(1)O(1)O(1).

Binary Search

We can use Binary Search to solve this problem.

Let the square root of xxx is kkk, k2≤xk^2 \leq xk2≤x. The lower bound is 000, and the upper bound is xxx. In each step, we need to compare the middle element mid2≤xmid^2 \leq xmid2≤x or mid2>xmid^2 > xmid2>x, adjust the range of the upper and lower bounds.

Since all our operations are integer operations, there is no error, so after getting the final answer ans\textit{ans}ans, there is no need to try ans+1\textit{ans} + 1ans+1 again.

Analysis

  • Time Complexity: O(log⁡x)O(\log x)O(logx)
  • Space Complexity: O(1)O(1)O(1).

Newton’s Method

The Newton’s Method is:

Let NNN be any number then the square root of NNN can be given by the formula:

root=0.5×(X+(N/X))root = 0.5 \times (X + (N / X)) root=0.5×(X+(N/X))

where XXX is any guess which can be assumed to be NNN or 111.

  1. In the above formula, XXX is any assumed square root of NNN and root is the correct square root of NNN.
  2. Tolerance limit is the maximum difference between XXX and root allowed.

But how to understand this method?

If we want to find the square root of the integer NNN. Obviously, the square root of NNN is the function y=f(x)=x2−Ny = f(x) = x^2 - Ny=f(x)=x2−N of zero.

We take any xnx_nxn​ as an initial value, at each iteration of the step. We find the point (xn,f(xn))(x_n, f(x_n))(xn​,f(xn​)) on the image of the function, make a slope through this point, the derivative of this point f′(xn)f'(x_n)f′(xn​), the intersection with the horizontal axis is denoted as xn+1x_{n+1}xn+1​ compared to xnx_nxn​. It’s closer to zero.

The equation of the straight line is:

2×xn=(xn2−N)/(xn−x)2 \times x_n = (x_n^2 - N) / (x_n - x) 2×xn​=(xn2​−N)/(xn​−x)

The intersection with the horizontal axis is the equation:

2×xn×x−(xn2+N)=02 \times x_n \times x - (x_n^2 + N) = 0 2×xn​×x−(xn2​+N)=0

which is the new iteration result xn+1x_{n+1}xn+1​:

xn+1=0.5×(xn+N/xn)x_{n+1} = 0.5 \times (x_n + N / x_n) xn+1​=0.5×(xn​+N/xn​)

After iterations, the value of the true zero point sqrt(N)sqrt(N)sqrt(N) is close enough to be an answer.

There comes two questions:

  1. Which initial value do we assign?
  2. When does the iteration end?

Initial value

As mentioned rule 1, we assign XXX to the NNN itself because we want to find the positive sqrt(N)sqrt(N)sqrt(N) and NNN is surely larger than sqrt(N)sqrt(N)sqrt(N).

Ending iteration

After each iteration, the answer is closer to the zero point. So when the intersection obtained from two adjacent iterations is very close, we can conclude that the result at this time is enough for us to get the answer.

As mentioned rule 2, if the calculated root comes inside the tolerance allowed then break out of the loop.

In general, it can be judged whether the difference between the results of two adjacent iterations is less than a very small non-negative number.

Analysis

  • Time Complexity: O(log⁡x)O(\log x)O(logx)
  • Space Complexity: O(1)O(1)O(1).
