[LeetCode][242. Valid Anagram] 3 Approaches: HashMap, Sorting and Counting

发表于 2022-02-07 更新于 2024-03-05 分类于 Leetcode Waline： 阅读次数： 本文字数： 2.8k 阅读时长 ≈ 3 分钟

By Long Luo

This article is the solution 3 Approaches: HashMap, Sorting and Counting of Problem 242. Valid Anagram.

Here shows 3 Approaches to slove this problem: HashMap, Sorting and Counting.

HashMap

t\textit{t}t is an anagram of s\textit{s}s which means that the characters in both strings appear in the same kind and number of times.

We can use two HashMap\texttt{HashMap}HashMap to store the characters and the number of times, then compare the keys and values.

Analysis

  • Time Complexity: O(n)O(n)O(n)
  • Space Complexity: O(S)O(S)O(S), S=26S = 26S=26.

Sorting

t\textit{t}t is an anagram of s\textit{s}s is equal to “two strings sorted equal”. Therefore, we can sort the strings s\textit{s}s and t\textit{t}t first, then check whether the sorted strings are equal.

Analysis

  • Time Complexity: O(nlogn)O(nlogn)O(nlogn), sorting needs O(nlogn)O(nlogn)O(nlogn), comparing two arrays need O(n)O(n)O(n), so total is O(nlogn)O(nlogn)O(nlogn).
  • Space Complexity: O(logn)O(logn)O(logn), because sorting needs O(logn)O(logn)O(logn) space.

Counting

Since the string contains only 262626 lowercase letters, we can maintain a frequency array table\textit{table}table of length 262626.

Traverse the frequency of the characters in the record string s\textit{s}s, minus table\textit{table}table the corresponding frequency in table\textit{table}table, if table[i]<0\textit{table}[i] \lt 0table[i]<0, it means that t\textit{t}t contains an extra character that is not in s\textit{s}s, just return false\textit{false}false.

Analysis

  • Time Complexity: O(n)O(n)O(n)
  • Space Complexity: O(S)O(S)O(S), S=26S = 26S=26.
