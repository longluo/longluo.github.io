Leetcode 396. 旋转函数 题解。

方法一： 暴力法

思路与算法

很容易想到的方法是遍历，使用 222 个循环，很明显最多只能有 lenlenlen 次旋转，因为对称性，所以数组下标可能会 >=len>= len>=len，所以要取余 (i+j)mod  len(i+j) \mod len(i+j)modlen。

每次旋转更新最大值，时间复杂度为 O(N2)O(N^2)O(N2)。

复杂度分析：

  • 时间复杂度：O(N2)O(N^2)O(N2)，其中 NNN 是数组 nums\textit{nums}nums 的长度。

  • 空间复杂度：O(1)O(1)O(1)。

方法三： 前缀和 + 滑动窗口

思路与算法：

因为需要旋转数组，可以将 numsnumsnums 进行复制拼接，得到长度为 2n2n2n 的新数组，在新数组上任意一个长度为 nnn 的滑动窗口都对应了一个旋转数组。

考虑在窗口的滑动过程中，计算结果会如何变化，假设当前我们处理到下标为 [i,i+n−1][i, i+n-1][i,i+n−1] 的滑动窗口，当前结果为：

cur=nums[i]×0+nums[i+1]×1+...+nums[i+n−1]×(n−1)cur = nums[i] \times 0 + nums[i+1] \times 1 + ... + nums[i+n-1] \times (n-1) cur=nums[i]×0+nums[i+1]×1+...+nums[i+n−1]×(n−1)

当窗口往后移动一位，也就是窗口的右端点来到 i+ni+ni+n 的位置，左端点来到 i+1i+1i+1 的位置。

我们需要增加新右端点的值，即增加 nums[i+n]×(n−1)nums[i+n] \times (n-1)nums[i+n]×(n−1)，同时减去旧左端点的值，即减少 nums[i]×0nums[i] \times 0nums[i]×0，然后更新新旧窗口的公共部分 [i+1,i+n−1][i + 1, i + n - 1][i+1,i+n−1]。

不难发现，随着窗口的逐步右移，每一位公共部分的权值系数都会进行减一。

nums[i+1]×1+nums[i+2]×2+...+nums[i+n−1]×(n−1)nums[i+1] \times 1 + nums[i+2] \times 2 + ... + nums[i+n-1] \times (n-1) nums[i+1]×1+nums[i+2]×2+...+nums[i+n−1]×(n−1)

变为

nums[i+1]×0+nums[i+2]×1+...+nums[i+n−1]×(n−2)nums[i+1] \times 0 + nums[i+2] \times 1 + ... + nums[i+n-1] \times (n-2) nums[i+1]×0+nums[i+2]×1+...+nums[i+n−1]×(n−2)

因此，公共部分的差值为 ∑idx=i+1i+n−1nums[idx]\sum_{idx=i+1}^{i+n-1}nums[idx]∑idx=i+1i+n−1​nums[idx]，我们可以使用前缀和进行优化。

复杂度分析：

  • 时间复杂度：O(N)O(N)O(N)，其中 NNN 是数组的长度。

  • 空间复杂度：O(1)O(1)O(1)。
