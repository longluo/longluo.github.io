[LeetCode][295. Find Median from Data Stream] Two Heaps with the Follow Ups

发表于 2022-11-12 更新于 2023-11-14 分类于 Data Structures and Algorithms Waline： 阅读次数： 本文字数： 3.6k 阅读时长 ≈ 3 分钟

By Long Luo

This article is the solution Two Heaps with the Follow Up’s Solution of Problem 295. Find Median from Data Stream.

Intuition

We can simply use a ArrayList\texttt{ArrayList}ArrayList to record the number and sort the list, then we can easily get the median element of the list. However, the Time Complexity will be O(n2logn)O(n^2logn)O(n2logn) and the Space Complexity is O(n)O(n)O(n).

It surely will be TLE and we have to find a better solution.

Heap

We can use Two Priority Queues (Heaps) to maintain the data of the entire data stream.

The min Heap denoted as queueMin\textit{queueMin}queueMin is used to maintain the number num≤median\textit{num} \leq \textit{median}num≤median. The max Heap denoted as queueMax\textit{queueMax}queueMax is used to maintain the number num>median\textit{num} \gt \textit{median}num>median.

  • When the total number of data stream elements is Even: queueMax.size()=queueMin.size()\texttt{queueMax.size()} = \texttt{queueMin.size()}queueMax.size()=queueMin.size(), the dynamic median is queueMax.peek()+queueMin.peek()2\frac{\texttt{queueMax.peek()} + \texttt{queueMin.peek()}}{2}2queueMax.peek()+queueMin.peek()​;

  • When the total number of data stream elements is Odd: queueMin.size()=queueMin.size()+1\texttt{queueMin.size()} = \texttt{queueMin.size()} + 1queueMin.size()=queueMin.size()+1, the dynamic median is queueMin.peek()\texttt{queueMin.peek()}queueMin.peek().

When we try to add a new number num\textit{num}num to the Two Heaps, the cases can be as follows:

  • num≤max⁡{queMin}\textit{num} \leq \max \{\textit{queMin}\}num≤max{queMin}

We need to add this number to queueMin\textit{queueMin}queueMin. The new median will be less than or equal to the original median, so we may need to move the queueMin.peek()\texttt{queueMin.peek()}queueMin.peek() to queueMax\textit{queueMax}queueMax.

  • num>max⁡{queMin}\textit{num} \gt \max \{\textit{queMin}\}num>max{queMin}

We need to add this number to queueMax\textit{queueMax}queueMax. The new median will be greater than or equal to the original median, so we may need to move the queueMax.peek()\texttt{queueMax.peek()}queueMax.peek() to queueMin\textit{queueMin}queueMin.

Analysis

  • Time Complexity: O(nlogn)O(nlogn)O(nlogn)
  • Space Complexity: O(n)O(n)O(n)

Follow Ups

If all integer numbers from the stream are in the range [0, 100], how would you optimize your solution?

  • We can use a bucket with a length of 101101101. Each bucket stores the number of occurrences of each number, and records the total number of elements in the data stream. When searching for the median, calculate the number of the median, and then scan all buckets from front to back to get the answer.

If 99% of all integer numbers from the stream are in the range [0, 100], how would you optimize your solution?

  • As last question, we can still use buckets to store the data. Then use Two Pointers to maintain the median. For the number out of range, we can use two arrays to record the number which less than 000 or greater than 100100100. If the median is not in [0,100][0, 100][0,100], we can perform brute force to find it.
