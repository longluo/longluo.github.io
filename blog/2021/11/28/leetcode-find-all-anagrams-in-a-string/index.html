[LeetCode][438. 找到字符串中所有字母异位词] 滑动窗口：从HashMap，数组，再到统计字母数量之差

发表于 2021-11-28 更新于 2023-11-14 分类于 Data Structures and Algorithms Waline： 阅读次数： 本文字数： 5.5k 阅读时长 ≈ 5 分钟

By Long Luo

Leetcode 438. 找到字符串中所有字母异位词 题目如下：

看到这道题，首先想到方法就是：

  1. 统计字符串 p\textit{p}p 字母及其数量；
  2. 使用双指针 + 滑动窗口，如果当前窗口的不同字母种类及其数量和字符串 p\textit{p}p 字符串一致，将当前下标增加为答案；
  3. 扫描到字符串 s\textit{s}s 结束。

方法一：HashMap + 滑动窗口

思路与算法：

使用 HashMap\texttt{HashMap}HashMap 存储字符串 p\textit{p}p 的不同字母种类及其数量。

在字符串 s\textit{s}s 中构造一个长度为与字符串 p\textit{p}p 的长度相同的滑动窗口，并在滑动中维护窗口中每种字母的数量；当窗口中每种字母的数量与字符串 p\textit{p}p 中每种字母的数量相同时，则说明当前窗口为字符串 p\textit{p}p 的异位词。

复杂度分析：

  • 时间复杂度：O(m+(n−m)×Σ)O(m+(n-m) \times \Sigma)O(m+(n−m)×Σ)，其中 nnn 为字符串 s\textit{s}s 的长度，mmm 为字符串 p\textit{p}p 的长度，Σ\SigmaΣ 为所有可能的字符数。需要 O(m)O(m)O(m) 来统计字符串 p\textit{p}p 中每种字母的数量；需要 O(m)O(m)O(m) 来初始化滑动窗口；需要判断 n−m+1n-m+1n−m+1 个滑动窗口中每种字母的数量是否与字符串 p\textit{p}p 中每种字母的数量相同，每次判断需要 O(Σ)O(\Sigma)O(Σ)。因为 s\textit{s}s 和 p\textit{p}p 仅包含小写字母，所以 Σ=26=26\Sigma=26=26Σ=26=26。

  • 空间复杂度：O(Σ)O(\Sigma)O(Σ)。用于存储字符串 p\textit{p}p 和滑动窗口中每种字母的数量。

方法三：数组 + 滑动窗口

思路与算法：

每次对滑动窗口的检查都需要检查两个词频数组，复杂度为 O(26)O(26)O(26)，实际上，只关心两个数组是否完全一致，因此可以只维护一个词频数组 cnt\textit{cnt}cnt 来实现。

统计滑动窗口和字符串 p\textit{p}p 中每种字母数量的差，使用变量 diff\textit{diff}diff 来记录当前窗口与字符串 p\textit{p}p 中数量不同的字母的个数，并在滑动窗口的过程中维护它，当 diff=0\textit{diff}=0diff=0 即为我们要寻找的答案。
