[LeetCode][519. 随机翻转矩阵] 2种方法：暴力法 和 降维 + 哈希表

Leetcode 519. 随机翻转矩阵 题目如下：

这道题和 384. 打乱数组 类似，考察的都是随机数的有用，下面我们用暴力法和哈希表 222 种方法来解决这个问题。

方法一：暴力 + 随机数

思路与算法：

首先就想到的是新建一个矩阵 matrix\textit{matrix}matrix，调用 reset()\texttt{reset()}reset() 时重置 matrix\textit{matrix}matrix。

但是调用 flip()\texttt{flip()}flip() 呢？

  1. 生成一个 000 到 total\textit{total}total 的随机数 seed\textit{seed}seed；
  2. 如果当前数组元素值为 111，说明已经被调用过，需要重新生成一个随机数，返回上一步；
  3. 如果当前数组元素值为 000，对其置 111，返回当前位置。

  • 时间复杂度：

    • flip\texttt{flip}flip：O(mn)O(mn)O(mn)。
    • reset\texttt{reset}reset：O(mn)O(mn)O(mn)。

  • 空间复杂度：O(mn)O(mn)O(mn)，需要 mnmnmn 的空间来存储矩阵。

方法二：降维 + 哈希表记录交换

思路与算法：

方法一需要新建一个二维数组，而 mmm 和 nnn 都是 10410^4104，乘起来是 10810^8108，维护如此之大的数组是非常耗费内存和时间的。有没有什么方法可以优化呢？

注意到最多调用 100010001000 次 flip()\texttt{flip()}flip() 和 reset()\texttt{reset()}reset() 方法，那么有没有办法可以只记录已经用过的数，还记得 384. 打乱数组 中洗牌算法中是如何交换数的吗？

将矩阵转换为一个长度为 m×nm \times nm×n 的一维数组 map\textit{map}map，对于矩阵中的位置 (i,j)(i, j)(i,j)，它对应了 map\textit{map}map中的元素 map[i×n+j]\textit{map}[i \times n + j]map[i×n+j]，这样就保证了矩阵和 mapmapmap 的元素映射。在经过 m×n−km \times n - km×n−k 次翻转 flip\texttt{flip}flip 后，我们会修改 map\textit{map}map 与矩阵的映射，使得当前矩阵中有 m×n−km \times n - km×n−k 个 111 和 kkk 个 000。

使用一个 HashMap\texttt{HashMap}HashMap 存储那些 map\textit{map}map 中那些被修改了的映射。

对于一个数 xxx：

  1. 如果 xxx 不是 HashMap\texttt{HashMap}HashMap 中的一个键，那么它直接映射到最开始的 (x/n,xmod  n)(x/n, x \mod n)(x/n,xmodn)；

  2. 如果 xxx 是 HashMap\texttt{HashMap}HashMap 中的一个键，那么它映射到其在 HashMap\texttt{HashMap}HashMap 中对应的值。

实际运行中 HashMap\texttt{HashMap}HashMap 的大小仅和翻转次数成正比，因为每一次翻转操作我们会交换 map\textit{map}map 中两个元素的映射，即最多有两个元素的映射关系被修改。

    • reset\texttt{reset}reset：O(F)O(F)O(F)，其中 FFF 是在上一次 reset()\texttt{reset}()reset() 之后执行 flip()\texttt{flip}()flip() 的次数。

  • 空间复杂度：O(F)O(F)O(F)，其中 FFF 代表执行函数 flip()\texttt{flip}()flip() 的次数。
