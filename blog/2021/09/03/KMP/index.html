理解KMP算法：从入门到推导

发表于 2021-09-03 更新于 2022-09-28 分类于 Data Structures and Algorithms Waline： 阅读次数： 本文字数： 5.8k 阅读时长 ≈ 5 分钟

By Long Luo

之前虽然知道 KMP 算法，但是一直无法深入理解其原理，最近看了2篇文章：从头到尾彻底理解KMP（2014年8月22日版） 和 KMP算法教程 ，然后再实际写代码，终于对 KMP\textit{KMP}KMP 算法有了一定了解了，下面写一写我个人的学习过程。

这篇文章主要从我个人学习过程来叙述：

为了解决什么问题？

KMP\textit{KMP}KMP 算法是一种字符串匹配算法，可以在 O(n+m)O(n+m)O(n+m) 的时间复杂度内实现两个字符串的匹配。

所谓字符串匹配，是这样一种问题：

  1. 字符串 P\textit{P}P 是否为字符串 S\textit{S}S 的子串？
  2. 如果是，P\textit{P}P 出现在 S\textit{S}S 的哪些位置？

其中 S\textit{S}S 称为主串；P\textit{P}P 称为模式串。

最常见的就是经常要用的搜索功能，比如要在一大段文字中找到某个句子或者找到出现的全部位置。在这种场景下，要找的句子就是给定的模式 P\textit{P}P，而大段文字就是目标字符串 S\textit{S}S。

从暴力法开始

我们先从最直观的地方开始：

  1. 两个字符串 A,B\textit{A}, \textit{B}A,B 的比较？
  2. 如果 P\textit{P}P 是 S\textit{S}S 的字串，第一个出现的位置在哪里？

所谓字符串比较，就是问两个字符串是否相等。

最朴素的思想，就是从前往后逐字符比较，一旦遇到不相同的字符，就返回 False\textit{False}False；如果两个字符串都结束了，仍然没有出现不对应的字符，则返回 True\textit{True}True。

暴力解法复杂度分析

刚才我们成功实现了暴力算法，那么其时间复杂度如何？

记 nnn 为串 S\textit{S}S 的长度，mmm 为串 P\textit{P}P 的长度。

考虑“字符串比较”这个小任务的复杂度。最坏情况发生在：两个字符串唯一的差别在最后一个字符。这种情况下，字符串比较必须走完整个字符串，才能给出结果，因此复杂度是 O(len(str))O(len(str))O(len(str)) 的。

由此，不难想到暴力算法所面对的最坏情况：

主串形如“AAAAAAAAAAAAA…B”，而模式串形如“AAAAA…B”。每次字符串比较都需要付出 O(m)O(m)O(m) 次字符比较的代价，总共需要比较 nnn 次，因此总时间复杂度是 O(nm)O(nm)O(nm)。

那么如何改进呢？

信息熵冗余

我们很难降低字符串比较的复杂度（因为比较两个字符串，真的只能逐个比较字符）。因此，我们考虑降低比较的趟数。如果比较的趟数能降到足够低，那么总的复杂度也将会下降很多。

要优化一个算法，首先要回答的问题是“我手上有什么信息？”　我们手上的信息是否足够、是否有效，决定了我们能把算法优化到何种程度。请记住：尽可能利用残余的信息，是 KMP\textit{KMP}KMP 算法的思想所在。

很明显在暴力算法中，模式字符串每次都需要比较，非常复杂，那么这里面有没有优化的时间呢？

假设现在文本串 S\textit{S}S 匹配到 iii 位置，模式串 P\textit{P}P 匹配到 jjj 位置：

  • 如果 j=−1j = -1j=−1，或者当前字符匹配成功（即 S[i]==P[j]\textit{S}[i] == \textit{P}[j]S[i]==P[j]），都令 i++,j++i++, j++i++,j++，继续匹配下一个字符；

  • 如果 j!=−1j != -1j!=−1，且当前字符匹配失败（即 S[i]!=P[j]\textit{S}[i] != \textit{P}[j]S[i]!=P[j]），则令 iii 不变，j=next[j−1]j = next[j - 1]j=next[j−1]。也就是说模式串 P\textit{P}P 相对于文本串 S\textit{S}S 向右移动了 j−next[j]j - next[j]j−next[j] 位。换言之，当匹配失败时，模式串向右移动的位数为：失配字符所在位置 - 失配字符对应的next值，即移动的实际位数为：j−next[j]j - next[j]j−next[j]，且此值大于等于 111。

next\textit{next}next 数组中值的含义是当前字符之前的字符串中，最长的相同前缀后缀。

例如如果 next[j]=knext[j] = knext[j]=k，代表j之前的字符串中有最大长度为 kkk 的相同前缀后缀。

此也意味着在某个字符失配时，该字符对应的 nextnextnext 值会告诉你下一步匹配中，模式串应该跳到哪个位置（跳到 next[j]next[j]next[j] 的位置）。如果 next[j]next[j]next[j] 等于 000 或 −1-1−1，则跳到模式串的开头字符，若 next[j]=knext[j] = knext[j]=k 且 k>0k > 0k>0，代表下次匹配跳到 jjj 之前的某个字符，而不是跳到开头，且具体跳过了 kkk 个字符。

如何求Next数组？

字符串前缀和后缀

字符串前缀是指不包含最后一个字符的所有以第一个字符开头的连续子串，后缀是指不包含第一个字符的所有以最后一个字符结尾的连续子串。

例如：
比如说有一个长度为 555 字符串"ababc"，

其前缀有"a", "ab", "aba", "abab"；
后缀有"c", "bc", "abc", "babc", "ababc"。

那么

  • 长度为前 111 个字符的子串a，最长相同前后缀的长度为 000；

以此类推：

  • 长度为前 444 个字符的子串aaba，最长相同前后缀的长度为 111。
  • 长度为前 555 个字符的子串aabaa，最长相同前后缀的长度为 222。
  • 长度为前 666 个字符的子串aabaaf，最长相同前后缀的长度为 000。

获取Next数组过程

KMP

KMP\textit{KMP}KMP 的实现其实和 next\textit{next}next 数组求解是相同的，只是当前匹配到相同时，跳出循环。

因为 next\textit{next}next 的求解本质也是用自身去匹配自身。

KMP算法复杂度分析

  • 时间复杂度：O(n+m)O(n+m)O(n+m)，其中 nnn 是字符串 haystack\textit{haystack}haystack 的长度，mmm 是字符串 needle\textit{needle}needle 的长度。我们至多需要遍历两字符串一次。

  • 空间复杂度：O(m)O(m)O(m)，其中 mmm 是字符串 needle\textit{needle}needle 的长度。我们只需要保存字符串 needle\textit{needle}needle 的前缀函数。
