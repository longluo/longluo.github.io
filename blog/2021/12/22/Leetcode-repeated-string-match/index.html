[Leetcode][686. 重复叠加字符串匹配] 3种方法：暴力，字符串哈希，KMP算法

Leetcode 686. 重复叠加字符串匹配 题目如下：

分析

可以分为下面333种情况：

  1. A\textit{A}A ==B\textit{B}B，复制 111 次；

  2. lenA>lenBlenA > lenBlenA>lenB，直觉是只需要复制 222 次；

  3. lenA<lenBlenA < lenBlenA<lenB，首先至少将 A\textit{A}A 复制几次后长度 >=lenB>=lenB>=lenB，然后至多再复制 A\textit{A}A 一次。

下面简单证明下，也就是复制次数的下限和上限：

  1. 至少将字符串 A\textit{A}A 复制 NNN 次之后主串 S\textit{S}S，那么 lenS>=lenBlenS >= lenBlenS>=lenB ，才有可能匹配，至此我们得到了复制次数的下限：lenBlenA+1\frac{lenB}{lenA}+1lenAlenB​+1；

  2. 如果从主串 S\textit{S}S 中找到子串 B\textit{B}B，因此可以明确子串的起始位置，不会超过 lenAlenAlenA；因为如果起始匹配位置 >=lenA>= lenA>=lenA，那么说明在之前已经被匹配过了。

综上分析可知：

lenBlenA+1≤repeatedTimes≤lenBlenA+2\frac{lenB}{lenA} + 1 \le repeatedTimes \le \frac{lenB}{lenA} + 2 lenAlenB​+1≤repeatedTimes≤lenAlenB​+2

方法一：暴力

思路与算法：

首先想到的是暴力法。最开始没有去分析复制次数上下限时，使用了一个朴素的想法：

  1. B\textit{B}B 中含有 A\textit{A}A 中没有的字母，那么直接返回 −1-1−1；
  2. 最少复制次数：B\textit{B}B 中字母出现次数 A\textit{A}A 中字母次数的最大值。

使用 HashMap\texttt{HashMap}HashMap 存储字母出现次数。

  • 空间复杂度：O(26)O(26)O(26)。
